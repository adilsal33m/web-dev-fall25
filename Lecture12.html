Introduction to Laravel

Laravel is a modern PHP framework designed for building web applications quickly and elegantly.

  • Built on top of PHP
  • Follows MVC architecture
  • Provides routing, ORM (Eloquent), and Blade templating
  • Supports REST APIs and web apps

Setting up Laravel

You can start using Laravel by:

# Step 1: Install Laravel globally
composer global require laravel/installer

# Step 2: Create a new project
laravel new blog

# Alternatively, using Composer directly
composer create-project laravel/laravel blog

# Step 3: Start the development server
cd blog
php artisan serve

Starter Kits

Laravel offers official starter kits that simplify authentication and frontend setup for modern applications.

React - Build single-page apps using React 19, TypeScript, Tailwind, and shadcn/ui via Inertia.

Vue - Create modern Vue apps using Composition API, TypeScript, Tailwind, and shadcn-vue via Inertia.

Livewire - Build reactive Blade-based UIs with Livewire, Tailwind, and the Flux UI component library.

Composer

Composer is PHP's dependency manager. Laravel relies heavily on it to manage packages.

# Add a package
composer require guzzlehttp/guzzle

# Update dependencies
composer update

# Check installed packages
composer show

All dependencies are listed in composer.json.

5. Autoloading

Composer automatically handles class loading using the PSR-4 standard, so you never need to manually include files.

<?php
namespace App\Services;

class GreetingService {
    public function sayHello($name) {
        return "Hello, $name!";
    }
}
  
<?php
// In a controller
use App\Services\GreetingService;

$greet = new GreetingService();
echo $greet->sayHello('Adil');

Laravel automatically loads all classes inside the app/ directory using namespace conventions.

PSR-4 Autoloading

PSR-4 defines how namespaces map to directory structures. The mapping is defined in composer.json under the autoload section. 

{
  "autoload": {
    "psr-4": {
      "App\\": "app/",
      "Database\\Factories\\": "database/factories/",
      "Database\\Seeders\\": "database/seeders/"
    }
  }
}

This means that any class with the namespace App\Services is automatically loaded from app/Services.

6. Code Organization in Laravel

Laravel’s folder structure is designed for clarity, modularity, and scalability. Let’s explore the key directories.

app/

Contains the core application logic, including models, controllers, middleware, policies, and services.

app/
├── Console/
├── Exceptions/
├── Http/
│   ├── Controllers/
│   ├── Middleware/
│   └── Requests/
├── Models/
└── Providers/

Most of your PHP code lives here, organized by namespace under App\.

database/

Holds your migrations, model factories, and seeders that define and populate your database schema.

database/
├── factories/
├── migrations/
└── seeders/

Use php artisan migrate and php artisan db:seed to manage your database lifecycle.

config/

Stores configuration files for your application — such as database, cache, mail, and session settings.

config/
├── app.php
├── database.php
├── mail.php
└── cache.php

Access values using config('app.name').

routes/

Defines all your web, API, and console routes.

routes/
├── web.php       # Browser routes
├── api.php       # API routes
├── console.php   # Artisan commands
└── channels.php  # Event broadcasting channels

Each file maps URL patterns to controllers or closures.

resources/

Contains your frontend assets and Blade templates.

resources/
├── views/          # Blade templates
├── css/
├── js/
└── lang/           # Localization files

Files here are compiled with Vite for frontend performance and modularity.

vendor/

Contains all PHP dependencies installed via Composer.

vendor/
├── laravel/
├── symfony/
└── psr/

You should never manually edit files in this folder.

node_modules/

Holds JavaScript and CSS dependencies installed via npm.

node_modules/
├── tailwindcss/
├── vite/
└── axios/

Used for frontend builds managed by Vite.

Vite

Vite is Laravel’s default frontend bundler for fast builds and hot module replacement.

# Run development server
npm run dev

# Build for production
npm run build

It compiles your resources/js and resources/css files and injects them into your Blade templates.

.env File

The .env file stores environment-specific configuration such as database credentials, app keys, and mail settings.

# .env

APP_NAME="MyLaravelApp"
APP_ENV=local
APP_KEY=base64:randomgeneratedkey==
APP_DEBUG=true
APP_URL=http://localhost

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=myapp_db
DB_USERNAME=root
DB_PASSWORD=

MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=null
MAIL_PASSWORD=null

.env File

These values are loaded into Laravel using the env() helper and can be accessed via config() files. 

<?php
// Example: config/database.php
'host' => env('DB_HOST', '127.0.0.1'),

Keep the .env file private — never commit it to version control. Use .env.example for sharing defaults.

Artisan CLI

Artisan is Laravel’s command-line interface that automates common tasks such as code generation, database migrations, and caching.

# View all available commands
php artisan list

# Display help for a specific command
php artisan help migrate

Artisan helps streamline development and maintenance through expressive, easy-to-use commands.

Common Commands

Some of the most frequently used Artisan commands include:

php artisan serve           # Start the local development server
php artisan migrate         # Run all pending migrations
php artisan make:model User # Create a new model
php artisan make:controller UserController
php artisan tinker          # Interact with your app via REPL
php artisan route:list      # Display all registered routes

Use php artisan list anytime to explore available commands by category.

"make" Commands

The make: commands generate Laravel components automatically, ensuring consistent structure and naming conventions.

php artisan make:model Product -mcr
# -m: Create migration
# -c: Create controller
# -r: Make it a resource controller

php artisan make:middleware CheckRole
php artisan make:request StoreUserRequest
php artisan make:seeder ProductSeeder
php artisan make:event OrderPlaced
php artisan make:listener SendOrderNotification

This approach keeps your application modular and reduces repetitive setup work.

Database & Migrations

Artisan provides tools to manage your database schema and seed test data.

php artisan migrate           # Run all pending migrations
php artisan migrate:rollback  # Undo last migration batch
php artisan db:seed           # Seed the database
php artisan migrate:fresh --seed # Recreate database and seed it

These commands let you version-control database structure and automate seeding for testing or local development.

Cache & Config Management

Artisan provides commands to manage application cache, routes, and configurations efficiently. 

php artisan cache:clear        # Clear application cache
php artisan route:clear        # Clear route cache
php artisan config:cache       # Cache configuration for faster load
php artisan view:clear         # Clear compiled Blade templates
php artisan optimize           # Optimize framework performance

These are especially useful when deploying applications to production.

Custom Commands

You can create your own Artisan commands for repetitive or project-specific tasks.

Create a new command:

php artisan make:command GreetUser

Run it using:

php artisan app:greet

<?php
namespace App\Console\Commands;

use Illuminate\Console\Command;

class GreetUser extends Command
{
    protected $signature = 'app:greet';
    protected $description = 'Display a greeting message';

    public function handle()
    {
        $this->info('Hello! Welcome to Laravel!');
    }
}

Blade Templates

Blade is Laravel’s templating engine — simple, yet powerful, and compiles down to plain PHP. :contentReference[oaicite:0]{index=0}

Blade files use the `.blade.php` extension and are stored in `resources/views`.

Displaying Data & Escaping

Use `{{ }}` for escaped output (protected against XSS). 

{ $name }}}

To render unescaped HTML (be careful!), use `{!! !!}`. 

{!! $htmlContent !!}

Raw PHP & @use

Inline PHP is supported using `@php ... @endphp`.

@php
  $counter = 1;
@endphp

You can import classes or functions/constants with `@use` (and optionally alias them). 

@use('App\\Models\\Flight')
@use(function App\\Helpers\\format_currency, 'formatMoney')

Directives: Conditionals

Blade provides clean syntax for `if`, `unless`, `isset`, `empty`, etc. 

@if (count($records) === 1)
  One record
@elseif (count($records) > 1)
  Multiple records
@else
  No records
@endif

@unless (Auth::check())
  You are not signed in.
@endunless

Directives:Loops

Loops: `@foreach`, `@for`, `@while`, `@forelse`.

@foreach ($users as $user)
  
{{ $user->name }}

@endforeach

@forelse ($users as $user)
  
{{ $user->name }}

@empty
  
No users

@endforelse

Loop Info: $loop & Nested Loops

Inside loops, Blade provides a `$loop` variable with useful info: `first`, `last`, `index`, `count`, `parent`, etc.

@foreach ($items as $item)
  @if ($loop->first) First iteration @endif
  @if ($loop->last) Last iteration @endif
  Index: {{ $loop->index }}
@endforeach

In nested loops, use `$loop->parent` to access parent loop’s state.

Conditional Classes & Styles

Use `@class` directive to conditionally include CSS classes. 

@php
  $isActive = false;
  $hasError = true;
@endphp

<span @class([
  'p-4',
  'font-bold' => $isActive,
  'text-gray-500' => ! $isActive,
  'bg-red' => $hasError,
])></span>

Similarly, `@style` lets you conditionally set inline styles. 

<span @style([
  'background-color: red',
  'font-weight: bold' => $isActive,
])></span>

Attribute Directives

Blade offers directives for HTML attributes: `@checked`, `@selected`, `@disabled`, `@readonly`, `@required`.

<input type="checkbox" @checked(old('active', $user->active))>
<select name="version">
  @foreach ($product->versions as $version)
    <option value="{{ $version }}" @selected(old('version') == $version)>{{ $version }}</option>
  @endforeach
</select>
<button type="submit" @disabled($errors->isNotEmpty())>Submit</button>

Includes & @each

Use `@include` to embed subviews; data from parent is available. 

@include('shared.errors')
@include('view.name', ['status' => 'complete'])
@includeIf('view.name', ['status' => 'complete'])
@includeWhen($condition, 'view.name', ['status' => 'complete'])
@includeUnless($condition, 'view.name', ['status' => 'complete'])

Use `@each` for compact looping + include syntax.

@each('view.name', $jobs, 'job', 'view.empty')

Blade Component Slots

Slots let you inject custom content into components. Use {{ $slot }} for default slot content.

<!-- resources/views/components/layout.blade.php -->
<div class="p-6 border rounded-lg">
  <header>{{ $header ?? 'Default Header' }}</header>
  <main>{{ $slot }}</main>
</div>
<x-layout>
  <x-slot name="header">
    <h1>Dashboard</h1>
  </x-slot>

  <p>Welcome to your dashboard.</p>
</x-layout>

Blade Component Slots

You can define multiple named slots using <x-slot name="..."> and access them via variables.

To pass attributes like classes or IDs from component calls to their root element, use {{ $attributes }} merging: 

<div {{ $attributes->merge(['class' => 'p-4 rounded']) }}>
  {{ $slot }}
</div>

Blade Layouts & Template Inheritance

Layouts define a common structure for multiple pages using @extends, @section, and @yield.

You can also use @section and @show to define default replaceable blocks.

<!-- resources/views/layouts/app.blade.php -->
<!DOCTYPE html>
<html>
<head>
  <title>My App - @yield('title')</title>
</head>
<body class="bg-gray-50">
  <header>@include('partials.navbar')</header>

  <main class="container mx-auto mt-6">
    @yield('content')
  </main>
</body>
</html>
<!-- resources/views/home.blade.php -->
@extends('layouts.app')

@section('title', 'Home')

@section('content')
  <h1 class="text-2xl font-bold">Welcome Home</h1>
  <p>This page uses the main layout.</p>
@endsection
