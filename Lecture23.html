Why Deploy on Shared Server?

  • Cheapest hosting option — ideal for student or early-stage projects
  • Simple management through cPanel / Plesk (no Linux commands needed)
  • Good for small to medium websites with low–moderate traffic
  • No server administration required — security, updates, and backups often handled by host
  • Fast to set up — you can deploy a Laravel/PHP app within minutes

Moving Files: Upload Tool / File Manager

  • Most shared hosts provide a browser-based File Manager
  • Allows simple drag-and-drop upload of project files
  • Can upload a ZIP of your Laravel project and extract on server
  • Useful when you don’t have FTP or Git access
  • Fastest way to deploy small updates

Moving Files: FTP / SFTP

  • Use a client like FileZilla or WinSCP to connect to the server
  • Requires FTP/SFTP credentials from cPanel
  • Provides full access to server files (upload, delete, rename)
  • More reliable for large uploads or frequent updates
  • SFTP is preferred — encrypted connection for security

Moving Files: GitHub Deployment

  • Some hosting panels allow connecting a GitHub repository
  • You can set up auto-deployment: every push updates the site
  • Pulls only changed files → faster than full upload
  • Good for teams or structured development workflows
  • Requires a build step locally if you cannot run Composer/Node on the server

Setting Up the Database

  • Create a new MySQL database from cPanel (usually via “MySQL® Databases”) — this will hold all your application tables.
  • Create a dedicated DB user with a strong password — avoids using the default master account.
  • Assign the user to the database and grant **ALL PRIVILEGES** — allows Laravel to run migrations, insert data, and modify tables.
  • Update your .env file with:
    DB_DATABASE, DB_USERNAME, DB_PASSWORD — Laravel will connect using these values.
  • Test connection by running a migration (if you have shell access) or by loading the app in the browser.

Setting Up Domain / Subdomain

  • Add a domain or sub-domain in cPanel (e.g., app.example.com) — this creates a directory for your site on the server.
  • Point the domain’s document root to Laravel’s public/ folder — only public files should be accessible from the web.
  • If needed, move your Laravel project so that the public/ folder matches the domain root (or update the root path in cPanel).
  • Enable HTTPS using AutoSSL / Let’s Encrypt — ensures secure connections and avoids browser warnings.
  • Confirm the domain is resolving by opening it in the browser after DNS propagation.

Configuring the Laravel Project


cp .env.example .env
php artisan key:generate
  • Update DB credentials
  • Set APP_ENV=production
  • Set APP_DEBUG=false
  • Don't forget to run php artisan storage:link

Applying Optimizations


php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan optimize

Reduces response time by caching configuration, routes, and views.

No Shell Access?

  • Install dependencies locally → upload vendor/ folder
  • Upload node_modules/ build output (public/build)
  • Trigger artisan commands via temporary web routes

// routes/web.php
Route::get('/optimize', function () {
    Artisan::call('optimize');
});

⚠️ Delete these routes after running them!

Deployment on VPS

Full control over server environment (root access).

How Is It Different From Shared Hosting?

  • You configure everything yourself
  • Can install multiple apps
  • More secure and scalable
  • Higher performance
  • Requires Linux command-line knowledge

Deploying Using Containers

  • Encapsulates application + dependencies
  • Avoids “works on my machine” issues
  • Examples: Docker, Podman
  • Easier to ship / version apps

Challenges of Using Containers

  • Requires knowledge of Docker & Linux
  • Needs resource monitoring
  • Security hardening (images, network)
  • Backup & persistent storage management

Continuous Integration / Continuous Deployment

Automate → Test → Deploy without manual steps.

How Does CI/CD Work?

  • Developer pushes code to Git
  • CI pipeline runs:
  • installs dependencies
  • runs tests
  • builds optimized assets
  • CD pipeline deploys the build to server
